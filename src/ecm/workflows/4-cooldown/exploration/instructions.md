# Exploration Workflow

## Overview

Exploration is blue-sky time to prototype ideas that might become future pitches. No commitment, no pressure - just curiosity.

**Purpose:** Generate raw ideas and early validations that feed into future shaping.

## What is Exploration?

Exploration is:
- **Curiosity-driven** - "What if we..."
- **Time-boxed** - Short experiments
- **Low-pressure** - No commitment to ship
- **Idea-generating** - Feeds future shaping

Exploration is NOT:
- Full feature development
- Shaping (that's separate)
- Commitments to ship
- "Sneaking in" features

## Step 5: Share (Optional)

If the exploration is interesting:

1. **Demo** - Show what you found
2. **Discuss** - Get reactions
3. **Note interest** - Gauge appetite
4. **File for shaping** - If there's interest

## Exploration Principles

1. **Curiosity first** - Follow what's interesting
2. **Time-boxed** - Don't go down rabbit holes
3. **No commitment** - Exploration â‰  feature
4. **Document findings** - Future you will thank you
5. **Share selectively** - Not every exploration is gold

## What Happens Next?

After exploration:

- **Interesting + urgent:** Start shaping (SP workflow)
- **Interesting + not urgent:** File for later
- **Not interesting:** Archive and move on
- **Needs more exploration:** Schedule for next cool-down

## Completion

Exploration complete. Findings documented.

If this should become a pitch, next steps:
1. Shape Problem (SP)
2. Write Pitch (WP)
3. Present at Betting Table (BT)

Remember: Most explorations won't become pitches, and that's okay.
